# Requirements Document

## Introduction

The ElevenLabs Ambient Audio feature enhances the POV Podcast session experience by adding two layers of AI-generated audio atmosphere:

1. **Session Background Music** — a single piece of mild, ambient background music generated by ElevenLabs that matches the emotional tone and historical context of the active Scenario. It plays softly throughout the session and adapts when the overall emotional tenor of the conversation shifts significantly.

2. **Character Sound Effects** — short, persona-specific ambient sound effects generated by ElevenLabs that play softly in the background while a given Persona is speaking, reinforcing that character's environment and identity (e.g. distant artillery for a front-line soldier, hospital sounds for a field nurse).

Both audio layers are additive and non-intrusive: they sit beneath the primary TTS speech audio and must never compete with or obscure it. Users can independently control or mute each layer. The feature integrates with the existing `VoiceEngine` (client-side Web Audio API), the Convex backend, and the ElevenLabs Sound Effects and Music APIs.

---

## Glossary

- **Platform**: The POV Podcast web application as a whole.
- **Session**: A single interactive podcast playback instance in which the user listens to and optionally participates in a Scenario conversation.
- **Persona**: An AI-driven character representing a real or archetypal historical figure, with a defined personality, emotional backstory, and assigned ElevenLabs voice.
- **Scenario**: A historical event or topic that serves as the context for a podcast session.
- **Ambient Audio System**: The subsystem responsible for generating, caching, and playing background music and character sound effects during a Session.
- **Background Music Track**: A single looping or continuous ambient music audio clip generated by ElevenLabs Music API for a Scenario, matched to the Scenario's emotional tone and historical period.
- **Character Sound Effect**: A short ambient audio clip generated by ElevenLabs Sound Effects API that represents the sonic environment of a specific Persona (e.g. battlefield sounds, hospital ambience, crowd noise).
- **Emotional Tone Profile**: A structured descriptor derived from the Scenario's title, era, description, and the aggregate emotional states of all active Persona Agents, used as the prompt for music generation.
- **Sound Effect Prompt**: A short natural-language description of a Persona's sonic environment, derived from the Persona's historical role, geographic origin, and Scenario context, used as the prompt for sound effect generation.
- **Ambient Audio Layer**: One of the two independently controllable audio streams — Background Music Track or Character Sound Effect — that play beneath the primary TTS speech.
- **VoiceEngine**: The existing client-side TypeScript service (`src/lib/voiceEngine.ts`) that manages TTS audio playback via the Web Audio API.
- **Convex Backend**: The serverless backend that stores Scenarios, Personas, Sessions, and Dialogue history, orchestrates AI generation, and manages user authentication.
- **ElevenLabs Music API**: The ElevenLabs endpoint used to generate ambient background music from a text prompt.
- **ElevenLabs Sound Effects API**: The ElevenLabs endpoint used to generate short ambient sound effect clips from a text prompt.
- **Audio Ducking**: The automatic, temporary reduction of the Background Music Track volume when a Persona begins speaking, and its restoration when speech ends.
- **Ambient Volume Level**: A user-configurable gain value (0.0–1.0) applied to an Ambient Audio Layer independently of the primary TTS speech volume.
- **Ambient Audio Cache**: Server-side storage (Convex file storage) of generated audio clips keyed by Scenario ID or Persona ID, so that the same clip is not regenerated on every session start.

## Requirements

### Requirement 1: Background Music Generation

**User Story:** As a user, I want ambient background music that matches the emotional tone of the historical scenario I am listening to, so that the session feels more immersive and atmospherically grounded.

#### Acceptance Criteria

1. WHEN a Session starts for a Scenario that does not yet have a cached Background Music Track, THE Ambient Audio System SHALL request a Background Music Track from the ElevenLabs Music API using an Emotional Tone Profile derived from the Scenario's title, era, description, and initial dialogue outline.
2. THE Ambient Audio System SHALL construct the Emotional Tone Profile as a natural-language prompt of between 20 and 120 words that describes the historical period, geographic setting, and dominant emotional atmosphere of the Scenario without referencing specific character names or plot details.
3. WHEN the ElevenLabs Music API returns a Background Music Track, THE Convex Backend SHALL store the audio file in Convex file storage and associate it with the Scenario ID so that subsequent Sessions for the same Scenario reuse the cached track.
4. WHEN a Session starts for a Scenario that already has a cached Background Music Track, THE Ambient Audio System SHALL use the cached track without making a new ElevenLabs Music API request.
5. IF the ElevenLabs Music API request fails or times out after 15 seconds, THEN THE Ambient Audio System SHALL proceed without background music for that Session and SHALL NOT block session start or dialogue playback.
6. THE Background Music Track SHALL be generated at a duration of between 60 and 180 seconds and SHALL loop seamlessly during the Session.
7. THE Ambient Audio System SHALL begin playing the Background Music Track within 3 seconds of the first dialogue turn's audio playback starting, so that music does not precede the conversation.

---

### Requirement 2: Character Sound Effect Generation

**User Story:** As a user, I want each character's speaking turns to be accompanied by a soft ambient sound effect that reflects their environment, so that I can feel the physical world each persona inhabits.

#### Acceptance Criteria

1. WHEN a Scenario is created or seeded and a Persona does not yet have a cached Character Sound Effect, THE Ambient Audio System SHALL request a Character Sound Effect from the ElevenLabs Sound Effects API using a Sound Effect Prompt derived from the Persona's historical role, geographic origin, Scenario era, and Scenario title.
2. THE Ambient Audio System SHALL construct the Sound Effect Prompt as a natural-language description of between 10 and 60 words that describes the ambient sonic environment of the Persona's physical location and role (e.g. "distant artillery fire and mud underfoot on a World War I trench, low rumble, non-melodic").
3. WHEN the ElevenLabs Sound Effects API returns a Character Sound Effect, THE Convex Backend SHALL store the audio file in Convex file storage and associate it with the Persona ID so that subsequent Sessions reuse the cached clip.
4. WHEN a Persona begins speaking during a Session, THE Ambient Audio System SHALL begin fading in the Character Sound Effect for that Persona within 500 milliseconds of the Persona's TTS audio starting.
5. WHEN a Persona finishes speaking during a Session, THE Ambient Audio System SHALL fade out the Character Sound Effect for that Persona within 500 milliseconds of the Persona's TTS audio ending.
6. IF the ElevenLabs Sound Effects API request fails or times out after 10 seconds for a given Persona, THEN THE Ambient Audio System SHALL proceed without a Character Sound Effect for that Persona and SHALL NOT block session start or dialogue playback.
7. THE Character Sound Effect clip SHALL loop while the Persona is speaking and SHALL stop looping when the Persona finishes speaking.
8. IF a Persona does not yet have a cached Character Sound Effect when the Persona's turn begins, THEN THE Ambient Audio System SHALL play the turn without a Character Sound Effect and SHALL generate and cache the sound effect in the background for future turns.

---

### Requirement 3: Volume Control and Audio Mixing

**User Story:** As a user, I want to independently control the volume of background music and character sound effects, so that I can customise the audio experience to my preference without affecting the primary speech audio.

#### Acceptance Criteria

1. THE Platform SHALL provide independent volume controls for the Background Music Track and Character Sound Effects in the Session Settings Sheet, each with a range of 0 (muted) to 100 (maximum ambient volume).
2. THE Platform SHALL provide a single toggle to mute or unmute all Ambient Audio Layers simultaneously, without changing the individual volume levels set by the user.
3. WHEN a user adjusts an Ambient Audio Layer volume control, THE Ambient Audio System SHALL apply the new gain level within 200 milliseconds without interrupting playback.
4. THE Ambient Audio System SHALL apply Audio Ducking to the Background Music Track when a Persona begins speaking: the Background Music Track volume SHALL reduce to 20% of its current Ambient Volume Level within 300 milliseconds of TTS audio starting.
5. WHEN a Persona finishes speaking, THE Ambient Audio System SHALL restore the Background Music Track to its full Ambient Volume Level within 500 milliseconds of TTS audio ending.
6. THE Ambient Audio System SHALL ensure that the combined output of all Ambient Audio Layers does not exceed a gain of 0.35 relative to the primary TTS speech audio at any point, so that speech remains clearly intelligible.
7. THE Platform SHALL persist the user's Ambient Volume Level preferences for Background Music and Character Sound Effects to the user's preferences record in the Convex Backend, and SHALL restore these preferences at the start of each subsequent Session.
8. WHEN a Session is paused, THE Ambient Audio System SHALL pause all Ambient Audio Layer playback immediately. WHEN the Session is resumed, THE Ambient Audio System SHALL resume all Ambient Audio Layer playback from their paused positions within 300 milliseconds.

---

### Requirement 4: Ambient Audio Cache Management

**User Story:** As a developer, I want generated ambient audio clips to be cached server-side, so that ElevenLabs API costs are minimised and session start times are fast.

#### Acceptance Criteria

1. THE Convex Backend SHALL store each generated Background Music Track as a file in Convex file storage, keyed by Scenario ID, and SHALL record the storage ID and a generation timestamp in the `scenarios` table.
2. THE Convex Backend SHALL store each generated Character Sound Effect as a file in Convex file storage, keyed by Persona ID, and SHALL record the storage ID and a generation timestamp in the `personas` table.
3. WHEN a cached Background Music Track or Character Sound Effect is requested, THE Convex Backend SHALL return a time-limited download URL for the stored file within 500 milliseconds.
4. THE Convex Backend SHALL regenerate a cached Background Music Track for a Scenario if the cached track is older than 90 days, replacing the old file in storage.
5. IF a stored audio file is missing or its download URL is unreachable, THEN THE Convex Backend SHALL trigger regeneration of that audio clip and SHALL return a null URL to the client so the Ambient Audio System proceeds without that layer until regeneration completes.
6. THE Convex Backend SHALL not generate duplicate audio clips for the same Scenario or Persona concurrently: IF a generation request is already in progress for a given Scenario ID or Persona ID, THEN THE Convex Backend SHALL wait for the in-progress request to complete rather than issuing a second ElevenLabs API call.

---

### Requirement 5: Accessibility and User Control

**User Story:** As a user with sensory sensitivities or hearing impairments, I want full control over ambient audio and clear labelling of audio controls, so that the feature does not create an inaccessible or uncomfortable experience.

#### Acceptance Criteria

1. THE Platform SHALL default both Ambient Audio Layers to a muted state for all users until the user explicitly enables them, so that ambient audio is opt-in rather than opt-out.
2. THE Platform SHALL provide ARIA labels for all Ambient Audio Layer volume controls and the global mute toggle, describing the control's purpose and current state (e.g. "Background music volume, currently 40%").
3. WHEN the global ambient audio mute toggle is activated, THE Platform SHALL update the ARIA label of the toggle to reflect the new state (e.g. "Unmute ambient audio") within one rendering cycle.
4. THE Platform SHALL ensure all Ambient Audio Layer controls are operable via keyboard navigation alone, consistent with the accessibility requirements of the existing Session interface.
5. THE Platform SHALL display a visible label identifying each volume control as "Background Music" or "Character Sounds" so that users can distinguish between the two Ambient Audio Layers without relying on audio cues alone.

### Requirement 6: Emotional Tone Adaptation

**User Story:** As a user, I want the background music to subtly reflect shifts in the conversation's emotional intensity, so that the audio atmosphere evolves with the dialogue rather than remaining static.

#### Acceptance Criteria

1. THE Ambient Audio System SHALL monitor the aggregate emotional state of all active Persona Agents after each Dialogue turn by computing a dominant mood from the distribution of current Persona moods.
2. WHEN the dominant mood of the Session changes from one mood category to a different mood category and has remained in the new category for at least 3 consecutive Dialogue turns, THE Ambient Audio System SHALL classify this as an Emotional Tone Shift.
3. WHEN an Emotional Tone Shift is detected, THE Ambient Audio System SHALL request a new Background Music Track from the ElevenLabs Music API using an updated Emotional Tone Profile that reflects the new dominant mood, and SHALL crossfade from the current track to the new track over 4 seconds.
4. THE Ambient Audio System SHALL cache the new Background Music Track in Convex file storage associated with the Scenario ID and the new dominant mood label, so that the same mood-specific track is reused if the same mood recurs in a future Session for the same Scenario.
5. IF the new Background Music Track request fails, THEN THE Ambient Audio System SHALL continue playing the existing track without interruption and SHALL NOT retry the request during the same Session.
6. THE Ambient Audio System SHALL NOT trigger more than one Emotional Tone Shift per 5 minutes of session time, to prevent excessive API calls and disruptive audio transitions.

---

### Requirement 7: Convex Backend Integration

**User Story:** As a developer, I want the ambient audio generation to be orchestrated by the Convex backend using the same patterns as existing ElevenLabs integrations, so that the feature is consistent with the existing architecture and maintainable.

#### Acceptance Criteria

1. THE Convex Backend SHALL expose a new internal action `generateBackgroundMusic(scenarioId, emotionalToneProfile)` that calls the ElevenLabs Music API, stores the result in Convex file storage, and updates the `scenarios` table with the storage ID and generation timestamp.
2. THE Convex Backend SHALL expose a new internal action `generateCharacterSoundEffect(personaId, soundEffectPrompt)` that calls the ElevenLabs Sound Effects API, stores the result in Convex file storage, and updates the `personas` table with the storage ID and generation timestamp.
3. THE Convex Backend SHALL expose a new query `getAmbientAudioUrls(scenarioId, personaIds)` that returns time-limited download URLs for the Background Music Track and all requested Character Sound Effects in a single round trip.
4. WHEN `generateBackgroundMusic` or `generateCharacterSoundEffect` is called for an ID that already has a valid cached file, THE Convex Backend SHALL return the existing storage ID without calling the ElevenLabs API.
5. THE Convex Backend SHALL store the ElevenLabs Music API key and Sound Effects API key as environment variables (`ELEVENLABS_API_KEY`, already in use) and SHALL NOT expose these keys to the client.
6. THE Convex Backend SHALL schedule `generateCharacterSoundEffect` for each Persona in a Scenario immediately after the Scenario is created or seeded, using `ctx.scheduler.runAfter(0, ...)`, consistent with the existing avatar generation pattern.

---

### Requirement 8: Round-Trip Audio URL Serialisation

**User Story:** As a developer, I want ambient audio URL records to serialise and deserialise correctly, so that cached audio references can be persisted and restored without data loss.

#### Acceptance Criteria

1. THE Convex Backend SHALL serialise each ambient audio record as a structured object containing the entity ID (Scenario ID or Persona ID), the Convex storage ID, the ElevenLabs generation prompt used, the dominant mood label (for music tracks), and the generation timestamp.
2. WHEN a serialised ambient audio record is deserialised, THE Convex Backend SHALL produce an object equivalent to the original in all fields.
3. FOR ALL valid ambient audio record objects, serialising then deserialising then serialising SHALL produce an identical JSON document (round-trip property).
4. IF a serialised ambient audio record is missing a required field, THEN THE Convex Backend SHALL return a descriptive validation error and SHALL NOT persist the incomplete record.
