# Design Document: ElevenLabs Ambient Audio

## Overview

The ElevenLabs Ambient Audio feature adds two independently controllable audio layers beneath the primary TTS speech in POV Podcast sessions:

1. **Background Music Track** — a looping ambient music clip generated by the ElevenLabs Music API, matched to the Scenario's emotional tone and historical period. It adapts when the conversation's dominant mood shifts significantly.
2. **Character Sound Effects** — short ambient clips generated by the ElevenLabs Sound Effects API, one per Persona, that play softly while that Persona is speaking to reinforce their physical environment.

Both layers are additive and non-intrusive. They sit beneath the primary TTS audio, support independent volume control, and are cached server-side in Convex file storage to minimise API costs and session start latency.

### Key Design Decisions

- **Server-side generation, client-side playback**: Audio is generated and cached by Convex actions (following the `generateAvatars`/`generateBanner` pattern), then served as time-limited URLs to the client. The client plays them via the Web Audio API, extending the existing `VoiceEngine`.
- **Opt-in by default**: Both layers default to muted (volume = 0) for all users, consistent with Requirement 5.1.
- **Graceful degradation**: Every generation failure is non-blocking. Sessions proceed without ambient audio rather than failing.
- **Idempotent generation**: A `generationStatus` field on each entity prevents duplicate concurrent API calls, mirroring the `avatarGenerationStatus` pattern.
- **ElevenLabs SDK**: The `@elevenlabs/elevenlabs-js` package is used for both Music and Sound Effects API calls, keeping the integration consistent and type-safe.

### Data Flow

1. **Scenario creation**: `generateScenario` schedules `generateCharacterSoundEffect` for each Persona (via `ctx.scheduler.runAfter(0, ...)`), and schedules `generateBackgroundMusic` for the Scenario.
2. **Session start**: `SessionPlayer` calls `getAmbientAudioUrls(scenarioId, personaIds)` to retrieve cached URLs. The `AmbientEngine` fetches and decodes the audio files.
3. **Playback**: `VoiceEngine` notifies `AmbientEngine` of TTS start/end events. `AmbientEngine` applies ducking, fade-in/out, and looping via the Web Audio API.
4. **Emotional tone shift**: After each dialogue turn, `SessionPlayer` computes the dominant mood from `personaAgentStates`. If a shift is detected, it calls `generateBackgroundMusic` with the new mood profile and crossfades to the new track.

## Components and Interfaces

### Backend: Convex Actions

#### `generateBackgroundMusic` (internal action)

```typescript
// convex/generateBackgroundMusic.ts
"use node";

export const generateBackgroundMusic = internalAction({
  args: {
    scenarioId: v.id("scenarios"),
    emotionalToneProfile: v.string(),
    moodLabel: v.optional(v.string()), // e.g. "frustrated" — for mood-keyed caching
  },
  handler: async (ctx, args): Promise<{ success: boolean; storageId?: Id<"_storage">; error?: string }> => { ... }
});
```

- Checks `scenarios.musicGenerationStatus` — if `"pending"`, returns early (idempotency guard).
- Sets status to `"pending"`, calls ElevenLabs Music API (`client.music.compose`), stores result in Convex file storage, updates `scenarios` table with `musicStorageId`, `musicGeneratedAt`, `musicMoodLabel`, and status `"complete"`.
- On failure: sets status to `"failed"`, returns `{ success: false }`.

#### `generateCharacterSoundEffect` (internal action)

```typescript
// convex/generateCharacterSoundEffect.ts
"use node";

export const generateCharacterSoundEffect = internalAction({
  args: {
    personaId: v.id("personas"),
    soundEffectPrompt: v.string(),
  },
  handler: async (ctx, args): Promise<{ success: boolean; storageId?: Id<"_storage">; error?: string }> => { ... }
});
```

- Checks `personas.sfxGenerationStatus` — if `"pending"`, returns early.
- Sets status to `"pending"`, calls ElevenLabs Sound Effects API, stores result, updates `personas` table with `sfxStorageId`, `sfxGeneratedAt`, and status `"complete"`.

#### `getAmbientAudioUrls` (public query)

```typescript
// convex/ambientAudioQueries.ts

export const getAmbientAudioUrls = query({
  args: {
    scenarioId: v.id("scenarios"),
    personaIds: v.array(v.id("personas")),
  },
  handler: async (ctx, args): Promise<AmbientAudioUrls> => { ... }
});

interface AmbientAudioUrls {
  musicUrl: string | null;
  sfxUrls: Record<string, string | null>; // personaId → URL
}
```

- Returns time-limited signed URLs from `ctx.storage.getUrl()` for all cached files in a single round trip.
- Returns `null` for any entity without a cached file.

### Frontend: `AmbientEngine`

New client-side class in `src/lib/ambientEngine.ts`:

```typescript
export interface AmbientAudioUrls {
  musicUrl: string | null;
  sfxUrls: Record<string, string | null>;
}

export interface AmbientVolumePrefs {
  musicVolume: number;   // 0–100
  sfxVolume: number;     // 0–100
  isMuted: boolean;
}

export class AmbientEngine {
  constructor(audioContext: AudioContext) {}

  async loadAudio(urls: AmbientAudioUrls): Promise<void>;

  startMusic(): void;
  stopMusic(): void;
  crossfadeToNewMusic(newUrl: string, durationMs?: number): Promise<void>;

  startSfxForPersona(personaId: string): void;
  stopSfxForPersona(personaId: string): void;

  duckMusic(): void;    // Reduce music to 20% over 300ms
  unduckMusic(): void;  // Restore music to full level over 500ms

  setMusicVolume(volume: number): void;   // 0–100
  setSfxVolume(volume: number): void;     // 0–100
  setMuted(muted: boolean): void;

  pause(): void;
  resume(): void;

  dispose(): void;
}
```

The `AmbientEngine` uses the same `AudioContext` instance as `VoiceEngine` to avoid multiple audio contexts. It manages:
- A `GainNode` for music volume (master + duck)
- A `GainNode` for SFX volume (per-persona)
- A `GainNode` for the global ambient ceiling (capped at 0.35 relative gain)
- `AudioBufferSourceNode` instances for looping playback

Renders:

### Frontend: Integration in `SessionPlayer`

`SessionPlayer` is extended to:
1. Call `useQuery(api.ambientAudioQueries.getAmbientAudioUrls, { scenarioId, personaIds })`.
2. Instantiate `AmbientEngine` alongside `VoiceEngine`.
3. Pass `onPlaybackStarted` / `onPlaybackComplete` callbacks from `VoiceEngine.playTurn` to `AmbientEngine.duckMusic()` / `AmbientEngine.unduckMusic()` and `AmbientEngine.startSfxForPersona()` / `AmbientEngine.stopSfxForPersona()`.
4. Compute dominant mood after each turn and call `AmbientEngine.crossfadeToNewMusic()` when a tone shift is detected.
5. Persist and restore `AmbientVolumePrefs` via a new Convex mutation/query on `userPreferences`.

## Correctness Properties

*A property is a characteristic or behavior that should hold true across all valid executions of a system — essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.*

### Property 1: Emotional Tone Profile word count

*For any* valid Scenario object (with title, era, timePeriod, description, and initialDialogueOutline), the string returned by `buildEmotionalToneProfile` SHALL contain between 20 and 120 words (inclusive).

**Validates: Requirements 1.2**

---

### Property 2: Sound Effect Prompt word count

*For any* valid Persona object (historicalRole, geographicOrigin) and Scenario object (era, title), the string returned by `buildSoundEffectPrompt` SHALL contain between 10 and 60 words (inclusive).

**Validates: Requirements 2.2**

---

### Property 3: Mute toggle preserves individual volume levels

*For any* pair of ambient volume levels (musicVolume ∈ [0, 100], sfxVolume ∈ [0, 100]), applying the global mute toggle and then applying it again (unmute) SHALL restore both volume levels to their original values unchanged.

**Validates: Requirements 3.2**

---

### Property 4: Audio ducking calculation

*For any* ambient music volume level v ∈ [0.0, 1.0], the ducked volume computed by `computeDuckedVolume(v)` SHALL equal `v * 0.20`, and restoring the volume after ducking SHALL return exactly `v`.

**Validates: Requirements 3.4, 3.5**

---

### Property 5: Combined ambient gain ceiling

*For any* music volume m ∈ [0.0, 1.0] and SFX volume s ∈ [0.0, 1.0], the combined ambient gain computed by `computeCombinedAmbientGain(m, s)` SHALL NOT exceed 0.35.

**Validates: Requirements 3.6**

---

### Property 6: Cache staleness check

*For any* generation timestamp `generatedAt` and current time `now`, `isCacheStale(generatedAt, now)` SHALL return `true` if and only if `(now - generatedAt) > 90 * 24 * 60 * 60 * 1000` milliseconds.

**Validates: Requirements 4.4**

---

### Property 7: Dominant mood computation

*For any* non-empty array of Persona moods, `computeDominantMood(moods)` SHALL return the mood that appears most frequently in the array. If two moods tie, the function SHALL return one of the tied moods consistently (deterministic tie-breaking).

**Validates: Requirements 6.1**

---

### Property 8: Emotional tone shift detection and rate limiting

*For any* mood history array and current mood, `detectEmotionalToneShift(history, currentMood)` SHALL return `true` if and only if the last 3 entries in the history are identical to `currentMood` and differ from the entry immediately preceding them. Additionally, *for any* sequence of tone shift timestamps, no two consecutive shift timestamps SHALL be within 5 minutes (300,000 ms) of each other.

**Validates: Requirements 6.2, 6.6**

---

### Property 9: Ambient audio URL aggregation completeness

*For any* array of `personaIds`, the object returned by `getAmbientAudioUrls(scenarioId, personaIds)` SHALL contain exactly one `sfxUrls` entry per personaId in the input array (the value may be `null` if no cached file exists), plus one `musicUrl` entry for the scenario.

**Validates: Requirements 7.3**

---

### Property 10: Ambient audio record serialisation round-trip

*For any* valid `AmbientAudioRecord` object, serialising to JSON and then deserialising SHALL produce an object that is deeply equal to the original in all fields (`entityId`, `entityType`, `storageId`, `generationPrompt`, `moodLabel`, `generatedAt`). Furthermore, serialising → deserialising → serialising SHALL produce an identical JSON string.

**Validates: Requirements 8.2, 8.3**

---

### Property 11: Ambient audio record validation rejects incomplete records

*For any* `AmbientAudioRecord` object with one or more required fields (`entityId`, `entityType`, `storageId`, `generationPrompt`, `generatedAt`) set to `undefined` or `null`, `validateAmbientAudioRecord(record)` SHALL return a non-null error object and SHALL NOT return a valid record.

**Validates: Requirements 8.4**

---

## Error Handling

### ElevenLabs API Failures

| Scenario | Behaviour |
|---|---|
| Music API timeout (>15s) | Set `musicGenerationStatus = "failed"`, return `null` music URL to client, session proceeds without music |
| SFX API timeout (>10s) | Set `sfxGenerationStatus = "failed"`, return `null` SFX URL for that persona, turn proceeds without SFX |
| Music API 429 (rate limit) | Treat as failure; no retry during the same session |
| Music API `bad_prompt` | Log the error; fall back to a simplified prompt (era + timePeriod only, no title/description) and retry once |
| SFX API `bad_prompt` | Log the error; fall back to a generic ambient prompt for the era and retry once |
| Tone shift music request fails | Continue playing existing track; do not retry during the session (Req 6.5) |

### Convex Storage Failures

| Scenario | Behaviour |
|---|---|
| `ctx.storage.getUrl()` returns `null` | Trigger background regeneration; return `null` URL to client (Req 4.5) |
| `ctx.storage.store()` throws | Set generation status to `"failed"`; log error |

### Client-Side Audio Failures

| Scenario | Behaviour |
|---|---|
| `AudioBuffer` decode fails | Log error; skip that audio layer silently |
| `AudioContext` suspended | Call `audioContext.resume()` on next user gesture |
| Network fetch of audio URL fails | Log error; skip that audio layer silently |

### Idempotency Guard

Before calling any ElevenLabs API, the action checks the entity's generation status:
- If `"pending"`: return early without making an API call (prevents duplicate concurrent calls, Req 4.6).
- If `"complete"` and not stale: return existing `storageId` (Req 7.4).
- If `"failed"` or `null`: proceed with generation.

## Testing Strategy

### Unit Tests (example-based)

Located in `src/lib/__tests__/ambientEngine.test.ts` and `convex/lib/__tests__/ambientAudioPrompts.test.ts`:

- Verify `buildEmotionalToneProfile` produces a non-empty string for known scenario inputs.
- Verify `buildSoundEffectPrompt` produces a non-empty string for known persona inputs.
- Verify `computeDominantMood` returns the correct mood for specific arrays.
- Verify `detectEmotionalToneShift` returns `false` for fewer than 3 consecutive identical moods.
- Verify `isCacheStale` returns `false` for a timestamp 1 day ago and `true` for a timestamp 91 days ago.
- Verify `AmbientEngine.setMuted(true)` sets the master gain to 0 without changing stored volume levels.
- Verify `AmbientEngine.duckMusic()` reduces the music gain to 20% of its current level.
- Verify `validateAmbientAudioRecord` returns an error for a record missing `storageId`.

### Property-Based Tests (fast-check)

Located in `src/lib/__tests__/ambientAudioProperties.test.ts`:

Uses `fast-check` (already in `package.json`) with `numRuns: 100`.

Each property test is tagged with:
```
// Feature: elevenlabs-ambient-audio, Property N: <property text>
```

- **Property 1**: Generate random scenario objects; verify `buildEmotionalToneProfile` word count ∈ [20, 120].
- **Property 2**: Generate random persona+scenario objects; verify `buildSoundEffectPrompt` word count ∈ [10, 60].
- **Property 3**: Generate random volume pairs; verify mute→unmute round-trip preserves values.
- **Property 4**: Generate random volume levels; verify ducking = 20% and unduck restores original.
- **Property 5**: Generate random (m, s) pairs; verify combined gain ≤ 0.35.
- **Property 6**: Generate random timestamps; verify `isCacheStale` matches the 90-day threshold exactly.
- **Property 7**: Generate random mood arrays; verify `computeDominantMood` returns the most frequent mood.
- **Property 8**: Generate random mood histories; verify shift detection and rate-limiting logic.
- **Property 9**: Generate random personaId arrays; verify URL map completeness.
- **Property 10**: Generate random `AmbientAudioRecord` objects; verify JSON round-trip equality.
- **Property 11**: Generate random records with one required field removed; verify validation rejects them.

### Integration Tests

Located in `convex/lib/__tests__/ambientAudio.integration.test.ts` (using `convex-test`):

- Verify `generateBackgroundMusic` stores a file and updates `scenarios` table when called with a mock ElevenLabs response.
- Verify `generateCharacterSoundEffect` stores a file and updates `personas` table.
- Verify `getAmbientAudioUrls` returns correct URLs for pre-populated cache entries.
- Verify idempotency: calling `generateBackgroundMusic` twice concurrently results in only one API call.
- Verify `generateScenario` schedules `generateCharacterSoundEffect` for each persona.

### Accessibility Tests
